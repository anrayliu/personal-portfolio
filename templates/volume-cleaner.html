Volume Cleaner

Open-source kubernetes microservice built from the ground up to delete unused storage resources. Developed during employment at Statistics Canada.

Highlights

⚙️

Systems Design

Chose robust tech stack and utilized microservice-oriented architecture to reduce coupling and limit scope.

🧑‍💻🧑‍💻

Collaboration

Developed side-by-side with colleague Justin Zhang, following GitHub best practices.

📝️️

Documentation

Extensively documented development process and design choices as GitHub issues.

Technologies Used

Go
Written in Go-lang.
Kubernetes
Designed for deployments on kubernetes clusters.
Docker
Builds images that are pulled into clusters for deployment.
GitHub Actions
Robust CI/CD pipeline for testing, linting, building, pushing, and vulnerability scanning.
Bash
Used with Makefile for convenient commands, as well as for integration testing in the CI/CD pipeline.
ArgoCD
Manages deployments on kubernetes clusters.
Azure
Provides our cluster (AKS) and image repository (ACR), but volume cleaner was designed to be abstract and interoperable.
Docker Hub
Secondary image repository location.
